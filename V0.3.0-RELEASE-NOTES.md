## 🎯 核心特性

### 1. 配置驱动架构
- **YAML配置**: 所有工作流、技能、反模式均通过YAML声明
- **环境变量覆盖**: `NOVA_WORKFLOW_MAX_ITERATIONS` 格式
- **多环境支持**: dev/prod/test 环境配置

### 2. 工作流引擎
- **阶段式执行**: 7阶段调研 / 6阶段写作 / 6阶段代码
- **条件分支**: 基于置信度/字段条件的阶段跳过
- **反模式检查**: 自动检测并提示常见问题
- **执行历史**: SQLite持久化存储

### 3. LLM客户端
| 类型 | 描述 | 连接方式 |
|------|------|----------|
| OllamaLLM | 本地Ollama | http://localhost:11434 |
| OpenClawLLM | OpenClaw Gateway | http://localhost:8080 |
| MockLLM | 模拟测试 | 无需连接 |

### 4. 工具系统
- **web_search**: DuckDuckGo搜索
- **web_fetch**: 网页内容抓取
- **code_execute**: 安全Python代码执行

### 5. 协作系统
- **Lead/Sub**: 主从协作模式
- **Swarm**: 多Agent共识
- **Review**: 审查协作

## 🚀 快速开始

### 安装依赖
```bash
pip install aiohttp fastapi uvicorn beautifulsoup4
```

### 基本使用

```bash
# 查看统计
python -m nova_agent.v0_3.main stats

# 列出资源
python -m nova_agent.v0_3.main list

# 运行工作流
python -m nova_agent.v0_3.main workflow research "调研Python异步编程"

# 运行协作
python -m nova_agent.v0_3.main collab "实现用户认证系统"

# 运行工具
python -m nova_agent.v0_3.main tool web_search --query "Python教程"
```

### 配置LLM

编辑 `config/default/llm.yaml`:
```yaml
llm:
  type: ollama
  model: qwen2.5:7b
  ollama:
    base_url: http://localhost:11434
```

### 启动API服务

```bash
python -m nova_agent.v0_3.server
# 或
uvicorn nova_agent.v0_3.server:app --host 0.0.0.0 --port 8080
```

API端点:
- `GET /health` - 健康检查
- `GET /stats` - 统计信息
- `POST /workflow/run` - 运行工作流
- `POST /skill/run` - 运行技能
- `POST /collab/run` - 运行协作
