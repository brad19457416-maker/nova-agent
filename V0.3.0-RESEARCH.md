### 关键洞察

#### 1. 工作流分层是核心

**OMX的发现**:
```
clarify → plan → execute → verify
```

**mattpocock的发现**:
```
PRD interview → 计划 → issues → TDD → verify
```

**impeccable的发现**:
```
audit → normalize → polish
```

**启发**: Nova Agent需要统一的工作流框架

#### 2. 技能需要结构化

**mattpocock技能模式**:
```yaml
skill-name:
  description: "做什么"
  steps:
    - step1
    - step2
  tools: [...]
```

**impeccable反模式**:
```
- Don't use Inter font
- Don't use purple gradients
- Don't nest cards in cards
```

**启发**: 技能应该包含"要做什么" + "不要做什么"

#### 3. 垂直切片思维

**mattpocock的prd-to-plan**:
- 不是全量实现
- 而是tracer-bullet垂直切片
- 每次只完成一个可验证的切片

**启发**: Nova Agent的写作工作流应该支持垂直切片

```python
class WorkflowFramework:
    """
    统一工作流框架
    参考: OMX + mattpocock
    """
    
    PHASES = [
        "clarify",    # 澄清需求
        "plan",       # 制定计划
        "execute",    # 执行
        "verify",     # 验证
        "polish"      # 打磨
    ]
    
    async def run(self, task, workflow_type="default"):
        if workflow_type == "research":
            return await self.research_workflow(task)
        elif workflow_type == "writing":
            return await self.writing_workflow(task)
        elif workflow_type == "code":
            return await self.code_workflow(task)
```

### 3. 垂直切片执行 (新增)

```python
class VerticalSliceExecutor:
    """
    垂直切片执行器
    参考: mattpocock/prd-to-plan
    """
    
    async def execute(self, task, slice_size="small"):
        """
        将大任务拆分为可独立验证的切片
        每个切片可独立测试/验证
        """
        slices = self.decompose(task, size=slice_size)
        
        results = []
        for slice_ in slices:
            result = await self.execute_slice(slice_)
            if not self.verify(result):
                return self.recover(result)
            results.append(result)
        
        return self.integrate(results)
```

## 💡 关键决策点

### 1. 工作流 DSL

**选项A**: YAML配置
```yaml
workflow: research
phases:
  - clarify
  - search
  - expand
```

**选项B**: Python装饰器
```python
@workflow("research")
async def research_phase(task):
    ...
```

**建议**: 选项A - 更易于用户定制

### 2. 技能分发

**选项A**: 内置核心技能
- 写作、代码、调研

**选项B**: 可安装技能包
- 像npm一样安装技能

**建议**: 选项A + 选项B - 核心内置，可扩展

### 3. 反模式来源

**选项A**: 手动编写
**选项B**: 从反馈中学习
**选项C**: AI自动生成

**建议**: 选项A开始 + 选项B迭代
