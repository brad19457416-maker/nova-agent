## 📋 核心决策清单

### 1. 产品定位决策

**问题**: Nova Agent要成为什么？

| 选项 | 描述 | 优势 | 劣势 |
|------|------|------|------|
| A. **通用框架** | 像LangChain，提供Agent基础设施 | 市场大，适用广 | 竞争激烈 |
| B. **深度研究专家** | 专注解决调研死板问题 | 差异化强，痛点明确 | 市场较小 |
| C. **写作助手** | 专注创意写作场景 | 与NC V4协同，垂直深入 | 通用性受限 |
| D. **代码助手** | 类似OMX，专注代码开发 | 市场需求大，付费意愿强 | 已有Claude/Cursor |

**我的建议**: **B为主 + C为辅**
- 核心差异化：深度研究引擎
- 优先场景：写作助手（利用NC V4积累）
- 次要场景：通用研究/代码开发

### 2. 工作流架构决策

**问题**: 如何设计工作流框架？

#### 2.1 架构层级

**选项A: 硬编码工作流**
```python
class ResearchWorkflow:
    async def run(self, task):
        await self.clarify()
        await self.search()
        await self.expand()
        await self.verify()
        await self.synthesize()
```
- ✅ 确定性强
- ❌ 不够灵活

**选项B: 配置驱动**
```yaml
workflow: research
phases:
  - name: clarify
    tool: deep_interview
  - name: search
    tool: web_search
  - name: expand
    tool: query_expansion
```
- ✅ 用户可定制
- ❌ 学习成本高

**选项C: 混合模式** ⭐推荐
```python
# 内置工作流
WORKFLOWS = {
    "research": ["clarify", "search", "expand", "verify"],
    "writing": ["outline", "draft", "review", "polish"],
    "code": ["understand", "plan", "implement", "test"]
}

# 允许用户扩展或覆盖
```

**你的选择？**

#### 2.2 阶段粒度

**问题**: 工作流阶段多细？

**粗粒度** (3-4阶段):
```
plan → execute → verify
```
- ✅ 简单易懂
- ❌ 不够精细

**细粒度** (6-8阶段):
```
clarify → plan → design → implement → test → review → polish
```
- ✅ 控制精确
- ❌ 可能过度工程

**我的建议**: **5-6个阶段**
```
1. clarify (澄清)
2. plan (规划)
3. execute (执行)
4. verify (验证)
5. polish (打磨)
6. deliver (交付)
```

### 4. 反模式系统

**问题**: 反模式有多重要？

#### 4.1 反模式作用

**impeccable的经验**:
- 明确告知AI"不要做什么"
- 比"要做什么"更有效
- 减少重复错误

#### 4.2 反模式来源

**选项A: 手动编写**
- 专家定义
- 质量高但更新慢

**选项B: 从反馈学习**
- 自动收集
- 数据驱动但可能有误

**选项C: AI生成**
- 自动生成
- 快速但不精确

**我的建议**: **A为主 + B为辅**
- 核心反模式：手动编写
- 扩展反模式：从用户反馈学习

**你的选择？**

#### 5.2 协作实现

**问题**: 如何实现Agent间协调？

**选项A: 共享内存**
- 所有Agent访问同一记忆
- 简单但可能有冲突

**选项B: 消息传递**
- Agent间通过消息通信
- 灵活但需要设计协议
